背景 / Problem
企业数据的真实现状
数据孤岛无处不在 — 格式各异、语义割裂、无法互联
💔 数据孤岛
销售数据在 Salesforce，财务数据在 Oracle ERP，
运营日志在 Kafka。
分析师每次做报告都需要手动 join CSV，耗时数天。
🔀 格式异构
同样是"客户"，CRM 叫 customer_id，
订单系统叫 cust_no，数仓叫 client_key。
没有统一语义，无法自动关联。
🧠 Palantir 的回答
把所有数据语义化成有类型、有关系的图谱（Ontology）。

分析师不再写 SQL，而是直接描述：
"给我所有高风险供应商及其关联合同"
⚡ 核心价值
数据在进入系统时就被语义化，
不是在出报告时才被理解。

Ontology = 企业的统一语言

核心流程 / Core Flow
数据源 → CanonicalRecord → Ontology Graph
三步把任意格式数据变成语义图谱节点
SourceAdapter trait
// 任何数据源只需实现这个 trait #[async_trait] pub trait SourceAdapter: Send + Sync { fn id(&self) -> &str; fn stream( &self, since: Option<Cursor>, // 增量同步 ) -> BoxStream<Result<CanonicalRecord>>; }
TomlMapping 规则（UI 自动生成）
entity = "Employee" [id] field = "emp_id" [map] name = "name|str" salary = "salary|float" dept_id = "dept_id|str|fk" # FK 自动建关系
OntologyEvent 输出流
OntologyEvent::Upsert { object: OntologyObject { id: "E001", entity_type: "Employee", attrs: { name: "Alice", salary: 120000.0 }, time: TimeBounds { valid_from, tx_time }, provenance: Provenance { source_id, .. } } } OntologyEvent::Link { from: "E001", to: "DEPT:Eng", rel: "BELONGS_TO", }
🔍 Discovery Engine — 3-Pass 自动关系发现
Pass 1 — 每条记录 → OntologyObject
Pass 2 — *_id 字段 → HAS 关系（FK 检测）
Pass 3 — 重复字符串值 → BELONGS_TO
无需手动定义关系，自动推断
计算 + 决策 / Logic & Workflow
图上计算 → 模式发现 → 工作流执行
从 Ontology 图中自动推导 Action，驱动可审计的业务流程
📐 Logic Actions — 图上纯函数计算
graph.objects_by_type("Employee")
→ 遍历 HAS 关系聚合 Transaction 总额
→ spend_ratio_pct(spend, salary)
→ expense_risk_level(ratio, concentration)
→ 输出风险标签：Low / Medium / High
🔗 Action Derivation — 关系模式 → 动作类型
BelongsTo → Logic: 按组聚合计算
HAS → Integration: Join 跨实体
LinkedTo → Workflow: 触发审批流
SimilarTo → Search: 相似模式匹配
⚙️ WorkflowRun — 可审计执行链
✓ Validate threshold → 超出 $3,200
✓ Notify dept head → 告警邮件已发
✓ Create hold → HOLD-ALICE-8200
✓ Archive audit → AR-00A2 写入日志

每步状态：Completed / Skipped / Failed
📊 Semantic Query Pipeline
Pipeline::new("high_risk") .step(Filter{ field:"amount", op:Gt, val:500.0 }) .step(Join{ right:employees, left_key:"employee_id" }) .step(Aggregate{ group_by:["department"], aggs:[Sum("amount"), Count], }) .run(transactions)

1
创建项目 — Projects 页面
点击 "＋ New Project"，输入项目名 → 自动跳转 Workspace。展示项目卡片点击直接进入。
2
上传 CSV — Step 1: Upload
拖拽上传 data/airline/ 目录下的 CSV 文件（flights, passengers, bookings 等）。
3
映射属性 — Step 2: Map Properties
配置 entity type、ID field、列→属性映射、FK 关系。点击 "Review & Build →" 自动持久化配置。
4
构建图谱 — Step 3: Build Ontology
点击 Build Ontology，观察进度动画。SourceAdapter 流式读取 → TomlMapping → OntologyEvent → 图写入。
5
查看图谱 — Step 4: View Graph
D3.js force-directed 图。关闭浏览器 → 重新打开项目 → 展示状态全部恢复（connector + mapping + graph）。
