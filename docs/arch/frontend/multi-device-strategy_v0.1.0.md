# 多设备策略：Web / Desktop / Pad / Mobile

> 版本：v0.1.0 | 日期：2026-03-19
> 关联：domain-separation_v0.1.0.md、ADR-16（前端选型）

---

## 一、核心原则

**功能随设备收敛，体验随设备优化。**

不同设备不是"同一个 UI 的缩小版"，而是针对该设备使用场景的**独立体验设计**：

```
Workstation / Desktop   → 生产力工具，功能完整，信息密度高
Laptop                  → 与 Desktop 基本一致，响应式收窄
iPad / Tablet           → 大多数功能可用，交互方式适配触摸
Mobile                  → 核心高频场景，随时随地，只读 + 快速操作
```

---

## 二、设备分级与断点

```
设备级别        屏幕宽度        主要场景
────────────────────────────────────────────────────────
Workstation     ≥ 1920px       数据工程师、开发者，多窗口并排操作
Desktop         1280 - 1919px  主力办公设备，完整功能
Laptop          1024 - 1279px  移动办公，偶尔需要 Function 编辑
Tablet (iPad)   768 - 1023px   会议中查数据、外出时处理审批
Mobile          < 768px        随时随地：问 AI、看通知、查状态
```

## 四、各设备级别的 UX 重点

### Workstation / Desktop（主力设备，信息密度最高）

```
┌──────┬─────────────────────────────┬────────────┐
│      │  主内容区                    │  右侧详情   │
│ Left │  (宽屏时可同时展示列表+详情) │  面板      │
│ Nav  │                             │  (可固定)  │
│      ├─────────────────────────────┴────────────┤
│      │  底部 Agent 输入条（常驻）               │
└──────┴──────────────────────────────────────────┘
```

- Monaco Editor 全尺寸展示，左侧文件树 + 中间编辑 + 右侧 Schema
- React Flow 画布有足够空间，节点不会拥挤
- 审计日志、数据表可以展示更多列
- 多标签页（Tab）支持同时打开多个对象详情

### Laptop（1024-1279px，略微收窄）

- LeftNav 默认收起为图标模式，Hover 展开
- 右侧详情面板改为覆盖层（Overlay），不压缩主区
- Monaco Editor 宽度自适应，Schema 面板折叠

### Tablet / iPad（768-1023px，触摸优先）

```
┌───────────────────────────────────┐
│  TopBar（简化，只留核心操作）       │
├───────────────────────────────────┤
│                                   │
│  主内容区（单列，触摸友好）         │
│  - 更大的点击目标（≥ 44px）        │
│  - 卡片视图替代密集表格             │
│  - 侧边栏改为底部 Sheet 或抽屉     │
│                                   │
├───────────────────────────────────┤
│  底部 Tab Bar（5个核心入口）        │
│  🏠首页  💬AI  🕸图  📂项目  🔔通知 │
└───────────────────────────────────┘
```

- Ontology 图只读浏览（支持双指缩放、单指拖拽）
- Function 编辑器不展示（Monaco 在 Touch 设备上体验差）
- 复杂的 Workflow 设计器不展示

### Mobile（< 768px，聚焦高频场景）

```
┌──────────────────┐
│  TopBar          │
│  [项目名 ▾]  🔔  │
├──────────────────┤
│                  │
│  主内容区（单列）  │
│                  │
│  场景 1：Agent   │
│  全屏对话界面     │
│                  │
│  场景 2：数据查看 │
│  卡片列表 + 搜索  │
│                  │
│  场景 3：通知    │
│  审批 / 提醒列表  │
│                  │
├──────────────────┤
│  底部 Tab Bar    │
│  💬  🔍  📂  🔔 │
└──────────────────┘
```

**Mobile 的 4 个核心场景（做好这 4 个就足够）：**

1. **AI 问答** — 随时提问，流式输出，和 Desktop 体验一致
2. **数据查找** — 搜索对象，查看核心字段（PII 字段仍然遵守权限）
3. **通知与审批** — 项目加入申请、Workflow 失败告警、摄入完成通知
4. **Workflow 状态** — 查看正在运行的 Workflow，紧急时可终止

## 五、Mobile 专属 UX 模式

### 下拉刷新 + 无限滚动
数据列表不用分页，下拉刷新最新数据，滚动到底自动加载下一页。

### Agent 语音输入（可选 P2）
移动端 Agent 输入框支持语音转文字（调用系统 Speech API），减少手机键盘输入负担。

### 推送通知（PWA 或 Native）
Workflow 失败、项目邀请等事件通过 Push Notification 推送到手机，点击直接跳转对应页面。

### 手势操作
- 左滑列表项 → 快捷操作（审批 / 拒绝）
- 右滑返回（iOS 体验一致性）
- 长按对象 → 底部 Action Sheet（分享、添加到项目、查看关系）

## 六、技术实现策略

### 响应式 Web（首选，覆盖全部设备）

**一套代码，CSS 断点驱动布局变化：**

```typescript
// 断点 Hook
const { isMobile, isTablet, isDesktop } = useBreakpoint();

// 按设备渲染不同组件
function DataView() {
  if (isMobile) return <DataCardList />;        // 卡片列表
  if (isTablet) return <DataCardGrid />;         // 卡片网格
  return <DataTable columns={ALL_COLUMNS} />;   // 完整表格
}

// 路由级保护：Mobile 禁止访问 Function 编辑页
function FunctionEditor() {
  if (isMobile) return <MobileUnsupported feature="Function 编辑器" />;
  return <MonacoEditor />;
}
```

**优点：** 一套代码库，维护成本低，SEO 友好，PWA 支持推送通知。
**缺点：** 部分复杂组件（Monaco、React Flow）在 Mobile 需要条件渲染，不能完全复用。

### Native App（可选，P2，Enterprise+ 场景）

如果大客户有强制要求（如 MDM 管理、离线缓存、生物识别登录），可在 Web 版稳定后开发 React Native App，复用 `@palantir/api-client` 包。

**不建议 Mobile 优先开发 Native App**，因为企业级应用核心价值在桌面端，Mobile 只是补充。

## 七、两个 App（Admin / Business）的设备支持范围

| App | Mobile | Tablet | Desktop+ |
|-----|:------:|:------:|:--------:|
| Admin App | — | ▶ 基础查看 + 紧急操作 | ✅ 完整 |
| Business App | ✅ 4 个核心场景 | ✅ 大部分功能 | ✅ 完整 |

**Admin App 不支持 Mobile：** 管理操作（模块配置、License 上传、部署变更）不应该在手机上操作，风险过高。Tablet 仅支持查看审计日志、查看模块状态、处理紧急审批。
