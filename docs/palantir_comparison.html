Palantir Foundry — 官方平台

企业级 / 完整
1
Data Connection — 原始数据接入
通过 Magritte 数据连接器统一接入各种数据源。支持增量同步（CDC）、连接器市场（100+ 预置连接器）、企业级鉴权（OAuth/SAML/Kerberos）。数据以原始状态落到 Raw 层，保留完整血缘。
Magritte Connectors S3 / HDFS / JDBC Kafka / Kinesis REST / GraphQL CDC 增量同步 100+ 预置连接器
部分
2
Foundry Data Lineage — 数据血缘
所有数据变换自动追踪血缘。每个 Dataset 记录从哪里来、经过哪些 Transform、最终去向哪里。支持跨项目血缘查询、Impact Analysis（改了某字段影响哪些下游）。
Dataset 版本化 Transform DAG Impact Analysis Provenance 溯源
缺失
3
Ontology — 语义对象层
Ontology 是 Foundry 核心。将清洗后的数据映射为有类型、有关系的语义对象（Object Types）。Object Type 定义属性、主键、关联关系。通过 Object Storage 持久化，支持亿级对象。
Object Type 定义 Link Type (关系类型) Object Storage Schema 版本管理 Ontology Editor UI
部分
4
Object Set & Filters — 语义查询
用 Object Set API 做语义查询：不写 SQL，而是描述"给我所有高风险员工"。支持 filter、groupBy、aggregate、navigate（沿关系遍历）。Object Set 是惰性求值，底层自动翻译为 Spark/SQL。
Object Set API Filter / GroupBy Aggregate Navigate (关系遍历) 惰性求值 / Spark 后端
部分
5
Functions — 计算层
Ontology Functions 是定义在对象类型上的计算逻辑（TypeScript / Python）。可以是 aggregation function（批量计算）或 action function（触发变更）。Functions 结果可以作为虚拟属性附加到对象上。
TypeScript Functions Python Transforms Derived Properties Aggregation Functions
部分
6
Actions — 写操作 / 状态变更
Actions 是对 Ontology 的写操作。通过声明式定义：哪些属性可以改、改完触发什么后续逻辑。Action 带完整的权限控制（谁可以触发）、审批流、变更通知。所有变更可追溯。
Action 声明式定义 RBAC 权限控制 Approval 审批流 Change Notification 完整审计日志
部分
7
Workflow Orchestration — 流程编排
基于 Ontology 事件触发多步骤工作流。Workflow 由触发条件 + 有序步骤组成，每步可调用 Action、发通知、调外部 API。与 AIP（AI Platform）集成，支持 AI 辅助决策。
Event-driven Trigger Step 编排 AIP 集成 (LLM) 外部 API 调用 SLA 监控
部分
8
AIP — AI Platform 决策辅助
AIP Logic 让 LLM 直接查询 Ontology（不是向量数据库）。AI Assistant 理解对象类型和关系，可以回答"哪些供应商有延迟风险"。Copilot 可以建议 Action、预测结果、生成报告。
AIP Logic (LLM + Ontology) AI Assistant Copilot 建议 RAG over Ontology
缺失
9
Workshop / Slate — 决策应用
低代码应用构建工具。Workshop 用于内部运营应用（数据看板 + Action 按钮）；Slate 用于分析报告（图表 + 过滤器）。两者都直接绑定 Ontology，数据实时同步。
Workshop (运营 App) Slate (分析报告) 低代码构建 实时 Ontology 绑定
缺失
VS

我们的实现 — 现状

Rust / Demo 阶段
1
SourceAdapter trait + CsvAdapter
定义了 SourceAdapter trait（stream / describe / cursor），已实现 CsvAdapter。
支持上传 CSV 文件，保存到 data/uploads/<project_id>/，读取 headers + 5行 sample 自动存入 SQLite。
✓ SourceAdapter trait ✓ CsvAdapter ✓ 文件持久化 ✗ DB / Kafka / HTTP Adapter ✗ CDC 增量同步 ✗ 连接器市场
2
数据血缘 — 基础结构存在，未完整
CanonicalRecord 有 source / ns / cursor / provenance 字段，理论上支持溯源。
但目前血缘链路未贯通：没有 Dataset 版本化，没有 Transform DAG，无 Impact Analysis。
~ CanonicalRecord.provenance ✗ Dataset 版本化 ✗ Transform DAG ✗ Impact Analysis
3
OntologyGraph + Mapping — 核心已实现
TomlMapping 将 CanonicalRecord 转为 OntologyEvent（Upsert / Link / Delete）。
UI 提供可视化 Map Properties（entity type / id field / 列→属性/类型/FK）。
mapping_config 持久化到 SQLite，重开项目自动恢复。
✓ OntologyObject + TimeBounds ✓ TomlMapping (声明式规则) ✓ UI 可视化配置 ✓ mapping_config 持久化 ✗ Schema 版本管理 ✗ Object Storage (仅 SQLite)
4
Pipeline 查询 — 结构有，未接 Ontology
palantir-pipeline 实现了 Filter / Join / Aggregate / Sort / Select 链式 Pipeline。
但目前 Pipeline 直接操作 Dataset（手动构建），未从 OntologyGraph 自动驱动。 live_ontology API 返回图数据，前端 D3.js 可视化。
✓ Pipeline (Filter/Join/Agg/Sort) ✓ D3.js 图可视化 ~ live_ontology API ✗ 从 Graph 直接语义查询 ✗ 惰性求值 / 大规模优化
5
Logic / Calculation — 已实现，未泛化
实现了 calc_salary_bands / calc_spend_metrics / calc_category_concentration。
逻辑写死在 Employee / Transaction 领域。缺少通用的"在任意 Object Type 上定义计算函数"机制。
✓ salary_band / spend_ratio / risk_level ✓ 图遍历计算（HAS 关系求和） ✗ 通用 Function 定义机制 ✗ Derived Properties (虚拟属性)
6
Action Derivation — 模式识别有，写操作未完整
derive_actions(graph) 扫描关系模式自动推断可执行动作（Logic/Integration/Workflow/Search）。
但 Action 定义是静态的，缺少声明式 Action Schema、权限控制、变更触发机制。
✓ 关系模式 → Action 推断 ✓ BelongsTo/HAS/LinkedTo/SimilarTo 映射 ✗ 声明式 Action 定义 ✗ RBAC 权限控制 ✗ 写操作变更追踪
7
WorkflowRun — 基础结构完整
WorkflowRun 有 trigger + 多步骤（Completed/Skipped/Failed）+ 审计记录。
但目前工作流是手动硬编码触发（run_high_spend_approval），缺少事件驱动自动触发机制。
✓ WorkflowRun 多步骤结构 ✓ StepOutcome 审计链 ✓ builds 表持久化构建记录 ✗ 事件驱动自动触发 ✗ SLA 监控 / 超时处理
8
AI 决策辅助 — Agent 骨架，未实装
palantir-agent 定义了 Agent<T: IngestTools> 和 IngestTools trait。
可以 handle_csv_ingest intent，调用 preview_csv / apply_csv。
但 Agent 未接入 LLM，无法做语义理解或 Ontology 问答。
~ Agent + IngestTools trait 骨架 ✗ LLM 接入 ✗ Ontology 语义问答 ✗ AI 建议 Action
9
决策应用 UI — 4步向导，基础可用
Web UI：项目管理 → 上传文件 → Map Properties → Build Ontology → D3 图可视化。
完成了 MVP 流程，但缺少运营级应用构建（动态报表、Action 按钮）和低代码能力。
✓ 项目管理（创建/列表/删除） ✓ 上传 + Map + Build + 可视化 ✓ 状态持久化（重开恢复） ✗ 动态报表 / 低代码 App ✗ Action 按钮绑定
